# Standard dashboards

Pre-built, ready-to-use dashboards that ship with Catalog Insights. They serve
two purposes:

1. **Out-of-the-box value** — each Analytics section opens with named standard
   dashboards (no setup), so a new user sees real governance metrics on day one.
2. **Enablement** — every file here is a valid `.studio.json` spec. Open one in
   the Designer to see exactly how a dashboard is shaped: panels, chart types,
   the queries they bind to, and the column bindings. They're the worked
   examples behind the format.

Each file conforms to `app/schema/dashboard.schema.json` and is regenerated by
`tools/build_dashboards.py`. `index.json` lists them per section.

| Section | Standard dashboards |
| ------- | ------------------- |
| Overview | Catalog health · Risk hotspots |
| System | Profiling health · Source inventory |
| User | Stewardship · Activity & ratings |
| Governance | Glossary coverage · Policy & lineage |
| Quality | Quality scores · DQ dimensions |
| Sensitivity | Exposure overview · PII discoveries |

These are starting points — duplicate one in the Designer and adjust, or
generate a new one with the AI drawer. Nothing here is locked.

> The `query` names (e.g. `trust_by_source`, `pii_discoveries`) refer to data
> accesses in the Query Library. The catalog of queries and how each maps to a
> PDC API call lives in `../../docs/DASHBOARDS.md` and `../../docs/PDC-CONNECTOR.md`.
